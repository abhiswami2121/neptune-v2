# Frontend Design — Production-Grade Interface Creation

Create distinctive, production-grade frontend interfaces with high design quality.

## When to Use

- Building web components from scratch
- Creating landing pages or marketing sites
- Designing dashboards and data displays
- Building React components with polished UI
- Styling/beautifying any web UI
- Creating HTML/CSS layouts

## Design Principles

1. **Distinctive** — Avoid generic AI aesthetics; every design should have character
2. **Production-Grade** — Code must handle all states (loading, empty, error, edge cases)
3. **Accessible** — WCAG 2.1 AA minimum; keyboard navigation, screen reader support
4. **Responsive** — Mobile-first; works from 320px to 4K displays
5. **Performant** — No layout shift, optimized images, minimal JavaScript

## Technology Stack

- **Tailwind CSS** — Utility-first styling
- **React 18+** — Component architecture
- **shadcn/ui** — Accessible component primitives
- **Framer Motion** — Animation library
- **Lucide Icons** — Consistent icon set

## Anti-Patterns to Avoid

- Purple gradients on everything (the "AI look")
- Centered text with no visual hierarchy
- Generic stock illustrations
- Over-engineered animations
- Ignoring mobile layout
- Missing focus states

## Output Format

Each design includes:
1. Component code (JSX/TSX)
2. Tailwind classes for styling
3. Animation specs (duration, easing, trigger)
4. Responsive breakpoints
5. Accessibility annotations
